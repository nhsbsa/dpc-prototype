Alpha

Off the back of Discovery, our 4 most riskiest assumptions that we have chosen to prioritise as a team within Alpha are; payment runs, data feeds, reporting and contractor's schedules.

Payment Runs

Pharmacy Payment Run

Hypothesis A

We believe that having the ability to preview data before starting each step of a payment run will be a useful feature for the Customer Payments Team as it will give them the confidence that the data being used is up to date. This will reduce the amount of additional manual checks that the team is currently performing.

Version 1

What we created:

  • Login
  • Dashboard
  • Payment Run
  • Payment Run (Monthly Breakdown)
  • Payment Run Steps
  • Payment Run Data Preview
  • Closing the Payment Run

What we plan to test:

  • Does the information architecture of the site make it easy to navigate, as well as adaptable for the future?
  • Does previewing certain items of key data give the team the confidence of continuing each payment run step?
  • When errors do happen, are the error messages more understandable and do they give the team the confidence to try running the step again depending on the severity?
  • Is the naming convention of the newly condensed payment run steps clear and understandable to the team?

What we found:

  • In principle, previewing key bits of data would work and reduce the number of times the CPT have to exit the payment run to run individual reports to check single figures.
  • Rather than add in the key dates for the following months payment run when closing the current one, the team often add / amend these dates in one go.
  • If there was any error message, the team would always request help from the support team - regardless of severity.

Any known issues?

  • Header and Footer links don't go anywhere
  • Only November 2021 payment run is available
